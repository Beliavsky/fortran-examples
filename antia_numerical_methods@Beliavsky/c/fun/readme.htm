NUMERICAL  METHODS  FOR  SCIENTIST S  AND ENGINEERS

C FUNCTIONS


This directory contains C Functions described in Appendix C. All functions use double precision (type double) floating point arithmetic.

All these functions have been tested to some extent, but there may be some bugs. Users are welcome to use these functions at their own risk. Any bugs discovered in these routines may be notified to

antia@tifr.res.in

For clarity most of the documents specify the name of functions using upper case letters, but in actual program all function names use only lower case letters.

These functions are simple translation of the corresponding Fortran subprograms. Since C does not permit use of complex arithmetic some functions to perform elementary operations on complex variables (CABS, CDIV, CSQRT) are also provided. The compiler may have an equivalent library routine to do these operations, though the arguments may be different. Further, the following Fortran subprograms requiring complex arithmetic have not been translated to C:

CONTUR, CROUT_C, DELVES, GAUBLK_C, GAUBND_C, GAUELM_C, GEVP_C, INVIT_C, INVIT_CC, INVIT_CL, LAGITC, NEWRAC, POLYC, SECANC, SECANC_2, SETMAT_C.
 

Each of these functions includes a declaration of the functions that are required by it and there is no need to include any header file. Nevertheless, all these declarations are included in the file antia.h which can be found in this directory. For general comments about C functions in the book the reader should refer to the first section of Appendix C, while more information about each of the functions can be found in subsequent sections of Appendix C. Examples illustrating the usage of these functions can be found in the directory c/xmp

During the revision, the names of the following two routines have changed

GAMLN   to   GAMMALN To calculate logarithm of Gamma function

RAN   to   RAN1 To generate random numbers with uniform distribution

Further the arguments of the following routines have changed

BSPFIT An additional argument to calculate the covariance matrix

LLSQ An additional argument to calculate the covariance matrix

POLFIT The 5th argument is now error in input data instead of weight (1/error**2)

RANGAU The seed is now real instead of integer

The random number generators RAN1 and RANGAU have been modified and hence the output of programs may not match the earlier version. Apart from this the calls to calloc where required have been modified to confirm to current standard using type size_t.

The following new routines have been added

betai.c, betap.c, betcon1.c, betcon.c, betser.c, fbeta.c, gammap.c, iranbin.c, iranpoi.c, linfitxy.c, pcor.c, shsort.c
