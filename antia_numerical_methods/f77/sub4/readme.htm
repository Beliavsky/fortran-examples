FORTRAN SUBPROGRAMS


This directory contains Fortran 77 subprograms described in Appendix B. All subprograms use single precision (REAL*4 / COMPLEX*8) arithmetic. Double and quadruple precision versions of these subprograms can be found in directories f77/sub8 and f77/sub16 respectively. Examples illustrating the usage of these subprograms can be found in the directory f77/xmp4

All these subprograms have been tested to some extent, but there may be some bugs. Users are welcome to use these subprograms at their own risk. Any bugs discovered in these routines may be notified to

antia@tifr.res.in

It may be noted that the single precision arithmetic uses 24 bits to represent the fraction part of floating point number giving a machine accuracy of about 6.0E-08. It can handle normalised floating point numbers in the range of approximately 1.17E-38 to 3.40E+38.

All constants in the subprograms have not been converted from double precision version and as a result some part of calculations may still be done in double precision. Although all subprograms have been converted to single precision, it may not be advisable to use them in single precision as it may be difficult to get the required accuracy. For example, SUBROUTINE REMES or GAUSWT are unlikely to succeed in single precision, except for problems requiring very small number of coefficients. On the other hand, in many problems, like interpolation, the inherent errors in input table may be so large that use of double precision may not improve the accuracy. Apart from accuracy the use of double precision arithmetic also extends the range of exponent. This extended range may be useful in extending the range of applicability of some functions like the Gamma function or high order Bessel functions. In single precision arithmetic these routines may give overflow in many cases.

On most machines, the difference in execution speed between single and double precision is not significant. Further, in many cases slight reduction in execution speed may be more than compensated by reduced number of iterations required when double precision arithmetic is used. In such cases, it is very likely that double precision version of program may execute faster than the single precision version.

During the revision, the names of the following two routines have changed

GAMLN   to   GAMMAL   To calculate logarithm of Gamma function

RAN   to   RAN1   To generate random numbers with uniform distribution

Further the arguments of the following routines have changed

BSPFIT   An additional argument to calculate the covariance matrix

LLSQ   An additional argument to calculate the covariance matrix

POLFIT   The 5th argument is now error in input data instead of weight (1/error**2)

RANGAU   The seed is now real instead of integer

The random number generators RAN1 and RANGAU have been modified and hence the output of programs may not match the earlier version.

The following new routines have been added

betai.f, betap.f, betcon1.f, betcon.f, betser.f, fbeta.f, gammap.f, iranbin.f, iranpoi.f, linfitxy.f, pcor.f, shsort.f
